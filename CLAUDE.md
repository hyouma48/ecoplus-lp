# エコプラス+ LP — Working Rules

このリポジトリで作業するすべてのエージェント・開発者が従うべきルール。
**実装作業に着手する前に、必ず本ファイルと `TEST_SPEC.md` を再読すること。**

---

## 1. テスト仕様書の取り扱い（最重要・絶対ルール）

- `TEST_SPEC.md` および `tests/` 配下の `*.spec.js` は、
  **「テストを合格させる」目的で書き換えてはならない。**
- テストが失敗したときに直すのは「実装側」であり、テスト側ではない。
- 仕様変更が真に必要な場合のフロー:
  1. 一度手を止め、ユーザーに変更理由を明示して相談する
  2. ユーザーの承認を得てから plan ファイルを更新
  3. その後で `TEST_SPEC.md` → `tests/` の順に変更
- 「テストの assertion を緩める」「セレクタを実装に合わせて変える」「
  required を外す」「期待値を書き換える」など、合格目当ての変更を見つけたら
  即座にロールバックすること。
- テスト要件と実装要件に乖離があると判断したら、勝手に判断せずユーザーに相談する。

## 2. 実装着手前の手順（必須）

1. `TEST_SPEC.md` を読み、当該タスクで満たすべき要件を確認
2. `SPEC.md` を読み、実装仕様を把握
3. 実装 → ローカルで `npx playwright test` 実行 → 全グリーン確認 → コミット

## 3. 広告・景表法上のルール

- 「No.1」「最高」「業界一」「絶対」「確実」など最大級・断定表現は、
  根拠表記なしで使わない。
- 「最短30分」「即日対応」など条件付き表現には注釈を入れる
  （例: `※ 交通状況・エリア・依頼内容により異なります`）。
- 架空のデータ（買取実績・お客様の声・スタッフ・メディア掲載）を使う箇所には、
  「※ サンプル表示」「※ イメージです」と明示する。
- 古物商許可番号などの法令表示は架空番号であることを明示。

## 4. ブランドガイドライン

- メインカラー: `#2A6FB0` 系（ロゴのブルー準拠）
- アクセントカラー: `#F39C12` 系（CTAボタン用の暖色）
- 背景: 白 + ライトグレー `#F7F9FC`
- ロゴ: ヘッダー左上 + フッターに必ず配置
- マスコット（パンダ）: ファーストビューに大きく + 各セクション導入で活用
- フォント: Noto Sans JP（Google Fonts、`font-display: swap`）

## 5. Git / 機密情報

- 実電話番号・実許可番号・実メールアドレスなど機密情報はコミットしない。
- 環境差分は `scripts/config.js` に集約し、本番値は別途 `config.local.js`
  などで上書きする方針。
- `.env` 系ファイルはすべて `.gitignore` 済み。

## 6. アクセシビリティ・パフォーマンス基準

- すべての `<img>` に `alt` 属性。装飾画像は `alt=""` でも可。
- `<html lang="ja">`。
- 見出しは `h1` 1個、以降 `h2` → `h3` 階層を守る。
- 折り畳み下の画像は `loading="lazy"`。
- Lighthouse モバイル: Performance 85+ / Accessibility 95+ / SEO 95+ を目標。

## 7. CTA 設計の固定ルール

- ファーストビュー内に `tel:` リンク必須。
- 全ページで `tel:` を3箇所以上、LINE 誘導を2箇所以上設置。
- 画面下に追従固定CTAバー（`#sticky-cta`）を必ず置き、電話・LINE・フォーム
  の3導線を含める。
- すべてのCTAクリックで `dataLayer.push` の計測イベントを発火させる。
